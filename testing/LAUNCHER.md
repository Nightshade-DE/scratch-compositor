# Stackcomp Launcher Guide

This document describes the runtime launcher behavior of
testing/stackcomp_run.

Use this guide for:
- release versus debug launcher modes
- runtime log location and shutdown list behavior
- managed portal resolution during session startup

## Portal Resolution

Portal startup is managed separately from user autostarts.

Resolution order in the current dev flow:

1. `config/startup.sh` sources `scripts/prepare_startup.sh`
2. Managed base file: `config/portals`
3. Optional user override: `~/.config/stackcomp/portals`

Behavior:

- `scripts/prepare_startup.sh` loads startup helpers and prepares nested/native runtime state before the user hook body runs.
- Native sessions load the managed base file before any session components are started.
- If the user override exists, it is sourced after the base file and may replace `stackcomp_start_portals()`.
- Nested sessions skip portal startup entirely.
- Users do not need a portal file unless they intentionally want to override the managed default behavior.

## Launcher-managed shutdown process list file (DO NOT EDIT THIS FILE!):

- shutdown_list.nfo

Path:

- $XDG_STATE_HOME/stackcomp/shutdown_list.nfo
- fallback: ~/.local/state/stackcomp/shutdown_list.nfo

This file is runtime state, not user configuration.
Do not edit it manually.

Short lifecycle:

1. At launcher start, the file is recreated/cleared.
2. Each service started via launch in config/startup.sh is appended to this file.
3. On shutdown, config/shutdown.sh reads this list and terminates the registered services (TERM, then KILL if needed).
4. Manual edits can desync this list and lead to unclean shutdown (leftover or wrongly terminated processes).

## Typical Runs

Release default:

  ./testing/stackcomp_run

Release-debug logs:

  STACKCOMP_DBG=1 ./testing/stackcomp_run

Debug + crash handler:

  STACKCOMP_DBG=2 ./testing/stackcomp_run

Alternate config path:

  STACKCOMP_CFG=/path/to/other.conf ./testing/stackcomp_run

Disable satellite:

  STACKCOMP_X11=0 ./testing/stackcomp_run

Force satellite display:

  STACKCOMP_X11_DISPLAY=:12 ./testing/stackcomp_run

Custom log directory:

  STACKCOMP_LOG_DIR=/tmp/stackcomp-test ./testing/stackcomp_run
